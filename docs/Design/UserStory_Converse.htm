Converse User Story

Ashon tells you “Do you know about converse mode?” > tell ashon No You tell Ashon “No” Ashon tells you “Type ‘converse ashon’ and type away!” > converse ashon You start conversing with Ashon. (!) Welcome to your first conversation. * Everything you type will now be relayed as a conversation instead of as a command. * Type ‘.end’ to stop conversing. * Type ‘.help converse’ for more detailed information. tell ashon> Testing You tell Ashon “Testing” tell ashon> Wow this is awesome! You tell Ashon “Why? Now I can bug you relentlessly!” Ashon tells you “Ha ha.” tell ashon> .end >

Additional notes:

  • Basically a special command processor mode which "falls back" to the regular command processor with input that started with "." (after stripping the "." for the fallback attempt).
  • Some edge cases to consider deeper might be that other special modes could potentially kick in, like after issuing a "dot command" which resulted in Paging mode or Y/N confirmations, etc. Perhaps the user just has to keep "dotting" those interactions like ".Y" to confirm, or ".end" to ineract and then go back to conversation mode.
  • Note that the prompt change from ">" serves as a continuous reminder that the user is in converse mode, since forgetting can result in sending what was intended as a command to the target. This could be especially helpful if they step away for a few minutes and forgot; their command attempts would be sent as commands.
  • Similarly, after idling for a minute or so: Perhaps the user could be warned and then automatically kicked out of converse mode.
  • If a player loses their connection, we could automatically inform anyone conversing at them (if they have permission to know this).
